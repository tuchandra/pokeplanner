# Pokopia Habitat Planner — Bootstrap & Reference UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a `pokeplanner` repo with bun + TypeScript + Vite, deploy a static SPA to `tusharc.dev/pokeplanner/` via GitHub Pages, and reproduce the layout of `pokehouseplan.com` (header controls, location tabs, house grid, Pokémon picker sidebar) with state persisted in `localStorage`.

**Architecture:** Static React SPA built with Vite. State managed with Zustand and persisted to `localStorage`. Game data (~300 Pokémon with habitats, work specialties, favorite tags, tastes; the location list; the trait catalog) is ingested once from the user's tracker TSV in `docs/reference/poketracker.tsv` into typed TypeScript modules under `src/data/`. Pokémon sprites are downloaded once into `public/sprites/<NNN>.png` and served from our own origin at `tusharc.dev/pokeplanner/sprites/...` rather than hotlinked. Drag-and-drop via `@dnd-kit/core` matches the reference site's interaction model. Deployed via GitHub Actions to GitHub Pages with custom domain `tusharc.dev` and base path `/pokeplanner/`.

**Capacity model (confirmed from live site + user):**
- Each location has a capacity of **40 points** (the X/40 counter in the location tabs is points used).
- House types contribute differently: **Prefab size 1 = 1 pt; Prefab size 2 = 2 pts; Prefab size 4 = 2 pts; Custom = 0 pts.** (Yes, prefab-4 contributes the same 2 pts as prefab-2.)
- The slot picker shows exactly three sizes: **1, 2, 4** (no 3). Same options for both Prefab and Custom; Custom houses just don't count against capacity.

**Tech Stack:**
- Runtime / package manager: `bun` (≥ 1.2)
- Language: TypeScript (strict)
- Type checker: `tsgo` via `@typescript/native-preview`
- Build: Vite (static SPA build, `base: '/pokeplanner/'`)
- UI: React 18 + functional components
- State: Zustand (`zustand/middleware/persist` for localStorage)
- Drag-and-drop: `@dnd-kit/core` (matches reference site's interaction model)
- Lint/format: Biome
- Tests: `bun:test` for pure logic; manual browser verification for UI
- Deploy: GitHub Actions → GitHub Pages

```ts
export type LocationId = 'WW' | 'BB' | 'RR' | 'SS' | 'PT';

export type Location = {
  readonly id: LocationId;
  readonly name: string;
  readonly capacity: number;
};

export type HouseType = 'prefab' | 'custom';

/** Slot counts: only 1, 2, or 4 are valid (no 3). Matches the three prefab sizes. */
export type SlotCount = 1 | 2 | 4;

/** Habitat lighting categories — confirmed exhaustive from reference bundle. */
export type Lighting = 'Bright' | 'Dark' | 'Dry' | 'Warm' | 'Humid' | 'Cool';

/**
 * Work specialty (drives in-game tasks: Grow makes plants grow faster, Build leads
 * building projects, Burn ignites flammables, etc.). '???' is the in-game placeholder
 * for legendaries/unknowns and is preserved verbatim. Source: reference bundle.
 */
export type Specialty =
  | 'Appraise' | 'Build' | 'Bulldoze' | 'Burn' | 'Chop' | 'Collect' | 'Crush'
  | 'DJ' | 'Dream Island' | 'Eat' | 'Engineer' | 'Explode' | 'Fly' | 'Gather'
  | 'Gather Honey' | 'Generate' | 'Grow' | 'Hype' | 'Illuminate' | 'Litter'
  | 'Paint' | 'Party' | 'Rarify' | 'Recycle' | 'Search' | 'Storage' | 'Teleport'
  | 'Trade' | 'Transform' | 'Water' | 'Yawn' | '???';

/** Six taste flavors. 'None' covers Pokémon with no taste preference (e.g. Ditto). */
export type Taste =
  | 'Sweet flavors' | 'Sour flavors' | 'Spicy flavors'
  | 'Bitter flavors' | 'Dry flavors' | 'None';

/**
 * Feature tags. The same vocabulary is used for two purposes:
 *  - a Pokémon's `favorites` (things it likes)
 *  - a house's `tags` (things present in the house)
 * Compatibility = overlap between the two sets.
 *
 * Stored as a string union — the canonical list lives in src/data/tags.ts.
 */
export type Tag = string;

export type Pokemon = {
  /** Stable, unique slug derived from name (e.g. 'bulbasaur', 'tatsugiri-curly-form'). */
  readonly id: string;
  /** Pokédex-like number from the tracker (zero-padded 3 chars). NOT unique — forms share. */
  readonly number: string;
  readonly name: string;
  readonly specialty1: Specialty;
  readonly specialty2: Specialty | null;
  readonly habitat: Lighting;
  readonly favorites: readonly Tag[];
  readonly taste: Taste;
  /** Path under the Vite base — e.g. '/pokeplanner/sprites/001.png' in prod. */
  readonly spriteUrl: string;
};

export type SlotIndex = 0 | 1 | 2 | 3;

export type House = {
  readonly id: string;
  name: string;
  type: HouseType;
  location: LocationId;
  slotCount: SlotCount;
  /** Pokémon ids by slot index, or null for empty. Length always equals slotCount. */
  slots: ReadonlyArray<string | null>;
  // Note: habitat lighting + feature tags are NOT stored. They're derived
  // from the assigned Pokémon (see derivedHabitats below).
};

/**
 * Capacity points contributed by a house against its area's 40-point budget.
 * Prefab-1 = 1, Prefab-2 = 2, Prefab-4 = 2, Custom = 0.
 */
export function capacityPoints(h: Pick<House, 'type' | 'slotCount'>): number {
  if (h.type === 'custom') return 0;
  return h.slotCount === 1 ? 1 : 2;
}

/**
 * Derived habitat chips for a house, computed from its assigned Pokémon.
 * Matches reference behavior:
 *   - lighting = unique set of `habitat` values across non-empty slots
 *   - tags    = intersection of `favorites` across all non-empty slots
 *   (so a tag is shown only when EVERY assigned Pokémon lists it)
 * Returns empty arrays when there are no assigned Pokémon.
 */
export function derivedHabitats(
  h: Pick<House, 'slots'>,
  lookup: (id: string) => Pokemon | undefined,
): { lighting: readonly Lighting[]; tags: readonly Tag[] } {
  const assigned = h.slots
    .map((id) => (id == null ? null : lookup(id) ?? null))
    .filter((p): p is Pokemon => p != null);
  if (assigned.length === 0) return { lighting: [], tags: [] };

  const lighting = Array.from(new Set(assigned.map((p) => p.habitat))) as Lighting[];

  let tagInter: Set<Tag> | null = null;
  for (const p of assigned) {
    const favs = new Set<Tag>(p.favorites);
    if (tagInter == null) tagInter = favs;
    else for (const t of tagInter) if (!favs.has(t)) tagInter.delete(t);
  }
  return { lighting, tags: Array.from(tagInter ?? new Set<Tag>()) };
}
```

```ts
/**
 * Minimal TSV parser. Treats the first non-empty row as the header. Trims
 * trailing tabs/whitespace per cell. Returns one record per data row, keyed
 * by header.
 */
export function parseTSV(input: string): ReadonlyArray<Record<string, string>> {
  const rows = input
    .split('\n')
    .map((line) => line.replace(/\r$/, ''))
    .filter((line) => line.length > 0);
  if (rows.length === 0) return [];
  const header = rows[0]!.split('\t').map((h) => h.trim());
  const records: Record<string, string>[] = [];
  for (let i = 1; i < rows.length; i++) {
    const cells = rows[i]!.split('\t');
    const rec: Record<string, string> = {};
    for (let j = 0; j < header.length; j++) {
      rec[header[j]!] = (cells[j] ?? '').trim();
    }
    records.push(rec);
  }
  return records;
}
```

```ts
#!/usr/bin/env bun
/**
 * Reads docs/reference/poketracker.tsv and writes
 * src/data/pokemon.generated.ts. Run from repo root: `bun run scripts/ingest-tracker.ts`
 *
 * Conversion rules:
 *  - id = slug(name)  e.g. "Mr. Mime" -> "mr-mime", "Tatsugiri Curly Form" -> "tatsugiri-curly-form"
 *  - number is taken verbatim from the "Number" column with the leading "#" stripped,
 *    zero-padded to 3 chars (some rows in the tracker have stray formatting like "#00")
 *  - habitat is title-cased (already is in the TSV)
 *  - favorites: trims, drops blanks, title-cases first letter of each word's first letter
 *    actually we lookup against the canonical TAGS set and lowercase compare to find the
 *    canonical casing — so "lots of nature" in the TSV maps to "Lots of nature" in TAGS
 *  - taste: TSV uses lowercase ("sweet flavors") — title-case the first word
 *  - specialty1/2: TSV uses "Ability 1/Ability 2" header — already canonical case in tracker
 *  - empty specialty2/taste/favorite cells map to null/[] respectively
 */
import { readFileSync, writeFileSync } from 'node:fs';
import { resolve } from 'node:path';
import { parseTSV } from '../src/lib/tsv';
import { TAGS } from '../src/data/tags';
import { TASTES } from '../src/data/tastes';
import { LIGHTING } from '../src/data/lighting';
import { SPECIALTIES } from '../src/data/specialties';

const ROOT = resolve(import.meta.dir, '..');
const SRC = resolve(ROOT, 'docs/reference/poketracker.tsv');
const OUT = resolve(ROOT, 'src/data/pokemon.generated.ts');

function slug(name: string): string {
  return name
    .toLowerCase()
    .replace(/[^a-z0-9]+/g, '-')
    .replace(/^-+|-+$/g, '');
}

function tagFromLowercase(s: string): string | null {
  if (!s) return null;
  const lower = s.trim().toLowerCase();
  for (const t of TAGS) if (t.toLowerCase() === lower) return t;
  // tolerate the "Slender Objects" / "Noise stuff" historical variants
  if (lower === 'slender objects') return 'Slender objects';
  if (lower === 'noise stuff') return 'Noisy stuff';
  return null; // caller decides whether to throw
}

function titleCaseFirst(s: string): string {
  if (!s) return s;
  return s[0]!.toUpperCase() + s.slice(1);
}

const tsv = readFileSync(SRC, 'utf8');
const rows = parseTSV(tsv);

type Out = {
  id: string;
  number: string;
  name: string;
  specialty1: string;
  specialty2: string | null;
  habitat: string;
  favorites: string[];
  taste: string;
};

const seen = new Set<string>();
const out: Out[] = [];
const errors: string[] = [];

for (const r of rows) {
  const name = r.Name;
  if (!name) continue;
  let number = (r.Number ?? '').replace(/^#/, '').padStart(3, '0');
  if (number === '00' || number === '000') number = '???'; // tracker has at least one stray "#00"
  const id = slug(name);
  if (seen.has(id)) {
    errors.push(`duplicate id: ${id}`);
    continue;
  }
  seen.add(id);

  const sp1 = r['Ability 1'] ?? '';
  const sp2 = r['Ability 2'] ?? '';
  if (!SPECIALTIES.includes(sp1 as never) && sp1) errors.push(`unknown specialty1: ${sp1} (${name})`);
  if (sp2 && !SPECIALTIES.includes(sp2 as never)) errors.push(`unknown specialty2: ${sp2} (${name})`);

  const habitat = r.Habitat;
  if (habitat && !LIGHTING.includes(habitat as never)) errors.push(`unknown habitat: ${habitat} (${name})`);

  const favs: string[] = [];
  for (const k of ['Favorites', 'Favorites_2', 'Favorites_3', 'Favorites_4', 'Favorites_5']) {
    // tracker has 5 unnamed columns after Habitat — parseTSV uses header row,
    // and they share the name "" or are blank. We'll re-extract by index instead:
  }
  // The tracker headers for the favorites are blank columns after "Habitat".
  // Re-read using positional access:
  const cells = Object.values(r);
  // header indices: 0=Number 1=Sprite 2=Name 3=Ability 1 4=Ability 2 5=Registered
  // 6=Home 7=Litter drop 8=Habitat 9..13=Favorites 14=Taste
  for (let i = 9; i <= 13; i++) {
    const v = cells[i] ?? '';
    if (!v) continue;
    const tag = tagFromLowercase(v);
    if (!tag) errors.push(`unknown favorite: "${v}" (${name})`);
    else favs.push(tag);
  }

  const tasteRaw = (cells[14] ?? '').trim();
  let taste = '';
  if (!tasteRaw) {
    taste = 'None';
  } else {
    const tCanon = titleCaseFirst(tasteRaw);
    if (TASTES.includes(tCanon as never)) taste = tCanon;
    else if (tasteRaw.toLowerCase() === 'none') taste = 'None';
    else errors.push(`unknown taste: "${tasteRaw}" (${name})`);
  }

  out.push({
    id,
    number,
    name,
    specialty1: sp1,
    specialty2: sp2 || null,
    habitat,
    favorites: favs,
    taste,
  });
}

if (errors.length) {
  console.error(`ingest produced ${errors.length} warnings:`);
  for (const e of errors.slice(0, 50)) console.error('  -', e);
  if (errors.length > 50) console.error(`  …and ${errors.length - 50} more`);
}

const banner = `// GENERATED by scripts/ingest-tracker.ts on ${new Date().toISOString()}\n// DO NOT EDIT BY HAND. Re-run: bun run scripts/ingest-tracker.ts\n`;
const body =
  banner +
  `import type { Pokemon } from '../types';\n\n` +
  `export const POKEMON_RAW: readonly Pokemon[] = ${JSON.stringify(
    out.map((p) => ({ ...p, spriteUrl: `/sprites/${p.number}.png` })),
    null,
    2,
  )} as const;\n`;

writeFileSync(OUT, body);
console.log(`wrote ${out.length} records to ${OUT}`);
```

```ts
#!/usr/bin/env bun
/**
 * Downloads sprites for every Pokémon in the generated roster into
 * public/sprites/<NNN>.png. Idempotent: skips files that already exist.
 *
 * Source URL is reconstructed from the tracker's number (zero-padded, 3 digits)
 * pointing at Serebii's Pokopia sprite directory. Run once, commit the PNGs,
 * then the deployed app serves them from our own origin.
 */
import { existsSync, mkdirSync, writeFileSync } from 'node:fs';
import { resolve } from 'node:path';
import { POKEMON_RAW } from '../src/data/pokemon.generated';

const ROOT = resolve(import.meta.dir, '..');
const OUT_DIR = resolve(ROOT, 'public/sprites');
mkdirSync(OUT_DIR, { recursive: true });

const numbers = Array.from(new Set(POKEMON_RAW.map((p) => p.number)));
console.log(`fetching ${numbers.length} unique sprites…`);

let downloaded = 0;
let skipped = 0;
let failed = 0;

for (const n of numbers) {
  const dest = resolve(OUT_DIR, `${n}.png`);
  if (existsSync(dest)) {
    skipped++;
    continue;
  }
  const url = `https://www.serebii.net/pokemonpokopia/pokemon/small/${n}.png`;
  try {
    const res = await fetch(url);
    if (!res.ok) {
      console.warn(`! ${n}: HTTP ${res.status}`);
      failed++;
      continue;
    }
    const buf = new Uint8Array(await res.arrayBuffer());
    writeFileSync(dest, buf);
    downloaded++;
    if (downloaded % 25 === 0) console.log(`  ${downloaded} downloaded…`);
  } catch (err) {
    console.warn(`! ${n}: ${(err as Error).message}`);
    failed++;
  }
}

console.log(`done — downloaded=${downloaded} skipped=${skipped} failed=${failed}`);
if (failed > 0) process.exit(1);
```

```ts
import type { House } from '../types';

export type PersistedState = {
  schemaVersion: 1;
  houses: House[];
};

const KEY = 'pokeplanner.v1';

export function readPersisted(): PersistedState | null {
  try {
    const raw = localStorage.getItem(KEY);
    if (!raw) return null;
    const parsed = JSON.parse(raw) as PersistedState;
    if (parsed?.schemaVersion !== 1) return null;
    return parsed;
  } catch {
    return null;
```

```ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';
import type { House, HouseType, LocationId, SlotCount, Specialty } from '../types';
import { STORAGE_KEY } from './storage';

type Filters = {
  activeLocation: LocationId;
  pendingType: HouseType;
  pendingSlots: SlotCount;
  pendingLocation: LocationId;
  /**
   * Work-specialty filter for the Pokémon picker. Multi-select via checkboxes
   * (matches reference). Empty array = no filter; otherwise show only Pokémon
   * whose specialty1 OR specialty2 is in this set.
   */
  specialtyFilter: readonly Specialty[];
  /**
   * Restrict the picker to Pokémon whose `habitat` is one of the selected
   * house's derived lighting chips. False = show all.
   */
  habitatCompatible: boolean;
  view: 'grid' | 'table';
};

type AppState = {
  houses: House[];
  filters: Filters;
  /** Currently focused house — drives the picker's habitat-compat filter. */
  selectedHouseId: string | null;
};

type AppActions = {
  addHouse: () => void;
  removeHouse: (id: string) => void;
  renameHouse: (id: string, name: string) => void;
  setSlotPokemon: (houseId: string, slot: number, pokemonId: string | null) => void;
  selectHouse: (id: string | null) => void;
  setFilter: <K extends keyof Filters>(key: K, value: Filters[K]) => void;
};

  activeLocation: 'WW',
  pendingType: 'custom',
  pendingSlots: 4,
  pendingLocation: 'WW',
  specialtyFilter: [],
  habitatCompatible: false,
  view: 'grid',
};

function newId(): string {
  return `h_${Math.random().toString(36).slice(2, 10)}`;
}

  persist(
    (set) => ({
      houses: [],
      filters: initialFilters,
      pendingSlot: null,

      addHouse: () =>
        set((s) => {
          const slotCount = s.filters.pendingSlots;
          const slots = Array<string | null>(slotCount).fill(null);
          const house: House = {
            id: newId(),
            name: 'House ' + (s.houses.length + 1),
            type: s.filters.pendingType,
            location: s.filters.pendingLocation,
            slotCount,
            slots,
          };
          return { houses: [...s.houses, house], selectedHouseId: house.id };
        }),

      removeHouse: (id) =>
        set((s) => ({ houses: s.houses.filter((h) => h.id !== id) })),

      renameHouse: (id, name) =>
        set((s) => ({
          houses: s.houses.map((h) => (h.id === id ? { ...h, name } : h)),
        })),

      setSlotPokemon: (houseId, slot, pokemonId) =>
        set((s) => ({
          houses: s.houses.map((h) => {
            if (h.id !== houseId) return h;
            const slots = [...h.slots];
            slots[slot] = pokemonId;
            return { ...h, slots };
          }),
          selectedHouseId: houseId,
        })),

      selectHouse: (id) => set({ selectedHouseId: id }),

      setFilter: (key, value) =>
        set((s) => ({ filters: { ...s.filters, [key]: value } })),
    }),
    {
      name: STORAGE_KEY,
      version: 1,
      partialize: (s) => ({ houses: s.houses, filters: s.filters, selectedHouseId: s.selectedHouseId }),
    },
  ),
);
```

```tsx
import { useState } from 'react';
import { SPECIALTIES } from '../data/specialties';
import { useStore } from '../state/store';
import type { Specialty } from '../types';

export function SpecialtyFilter() {
  const selected = useStore((s) => s.filters.specialtyFilter);
  const habitatCompat = useStore((s) => s.filters.habitatCompatible);
  const setFilter = useStore((s) => s.setFilter);
  const [open, setOpen] = useState(false);

  function toggle(s: Specialty) {
    const set = new Set(selected);
    if (set.has(s)) set.delete(s);
    else set.add(s);
    setFilter('specialtyFilter', Array.from(set));
  }

  const label = selected.length === 0 ? 'All Specialties' : `${selected.length} selected`;

  return (
    <div className="specialty-filter">
      <button
        type="button"
        className={`specialty-filter__trigger ${open ? 'is-open' : ''}`}
        onClick={() => setOpen((v) => !v)}
        aria-expanded={open}
      >
        <span>{label}</span>
        <span aria-hidden>▾</span>
      </button>
      {open && (
        <ul className="specialty-filter__menu" role="listbox" aria-multiselectable>
          {SPECIALTIES.map((s) => (
            <li key={s}>
              <label>
                <input
                  type="checkbox"
                  checked={selected.includes(s)}
                  onChange={() => toggle(s)}
                />
                {s}
              </label>
            </li>
          ))}
        </ul>
      )}
      <button
        id="habitat-filter-toggle"
        type="button"
        className={`habitat-toggle ${habitatCompat ? 'is-on' : ''}`}
        onClick={() => setFilter('habitatCompatible', !habitatCompat)}
        aria-pressed={habitatCompat}
      >
        Habitat: {habitatCompat ? 'compatible' : 'all'}
      </button>
    </div>
  );
}
```

- [ ] **Step 2: Write `PokemonPicker.tsx`** (draggable items via `@dnd-kit/core`)

```tsx
import { useDraggable } from '@dnd-kit/core';
import { POKEMON, POKEMON_BY_ID } from '../data/pokemon';
import { useStore } from '../state/store';
import { derivedHabitats, type Pokemon } from '../types';
import { SpecialtyFilter } from './SpecialtyFilter';

function PickItem({ p }: { p: Pokemon }) {
  const { attributes, listeners, setNodeRef, isDragging } = useDraggable({
    id: `pokemon:${p.id}`,
    data: { kind: 'pokemon', pokemonId: p.id },
  });
  return (
    <li
      ref={setNodeRef}
      {...attributes}
      {...listeners}
      className={`pick ${isDragging ? 'pick--dragging' : ''}`}
      title={`${p.name} — ${p.habitat}, ${p.specialty1}${p.specialty2 ? ` / ${p.specialty2}` : ''}`}
    >
      <img src={p.spriteUrl} alt={p.name} draggable={false} />
    </li>
  );
}

export function PokemonPicker() {
  const specFilter = useStore((s) => s.filters.specialtyFilter);
  const habitatCompat = useStore((s) => s.filters.habitatCompatible);
  const houses = useStore((s) => s.houses);
  const selectedHouseId = useStore((s) => s.selectedHouseId);

  // Compute the selected house's lighting set for the habitat filter.
  const selectedHouse = selectedHouseId
    ? houses.find((h) => h.id === selectedHouseId) ?? null
    : null;
  const targetLighting = selectedHouse
    ? new Set(derivedHabitats(selectedHouse, (id) => POKEMON_BY_ID.get(id)).lighting)
    : null;

  const visible = POKEMON.filter((p) => {
    if (specFilter.length > 0) {
      const ok = specFilter.includes(p.specialty1) ||
        (p.specialty2 != null && specFilter.includes(p.specialty2));
      if (!ok) return false;
    }
    if (habitatCompat && targetLighting && targetLighting.size > 0 && !targetLighting.has(p.habitat)) {
      return false;
    }
    return true;
  });

  return (
    <div className="picker">
      <div className="picker__head">
        <SpecialtyFilter />
      </div>
      <ul className="picker__grid">
        {visible.map((p) => <PickItem key={p.id} p={p} />)}
      </ul>
    </div>
  );
}
```

## Phase 5 — Custom UI tweaks (post-bootstrap; informs Phase 4 design)

These are user-requested deviations from the reference layout. They are NOT in scope for the bootstrap milestone (Phases 1–4 ship the reference UI faithfully), but the Phase 4 components should be designed so these can be added without rewriting them.

### Class A — Richer Pokémon detail

A1. **Group picker by skill/specialty.** A toggle (or a separate view mode) on the picker that groups Pokémon by `specialty1`. Should NOT reorder the default flat list — that's used to add evolutionary lines together. *Phase-4 design hook:* keep `PokemonPicker` rendering logic in a single function that receives `visible: Pokemon[]` and a `groupBy: 'none' | 'specialty'`; default `'none'`.

A2. **Pokémon detail panel.** Clicking a Pokémon (separate from drag) opens a panel showing: name, sprites, habitat, specialty1+2, taste, full `favorites` list, and "similar Pokémon" (same specialty1 OR ≥3 shared favorites). *Phase-4 design hook:* the picker item should accept an `onClick` that doesn't conflict with drag — dnd-kit's `activationConstraint: { distance: 6 }` already prevents short clicks from starting a drag, so a click handler on the item works.

A3. **Litter-drop items.** Litter-specialty Pokémon drop items in the game; the user's tracker has a `Litter drop` column (currently empty in the TSV but reserved for future fill-in). The detail panel from A2 should surface this when set, and the picker can have a Litter-drop filter mode. *Phase-4 hook:* `Pokemon.litterDrop?: string` optional field on the type; ingest reads the column.

B1. **Recommendations on selection, not reorder.** When a slot in a house is the active drag target (or the house is selected with the habitat-compat toggle on), surface a small "Recommended" section at the top of the picker (e.g. top 6 candidates by compatibility score) without reordering the main list. Matches user preference: keeping evolutionary neighbors visually adjacent is more important than ranking. *Phase-4 hook:* `PokemonPicker` renders an optional `<Recommended>` block above the main grid; the main grid stays in its default order.

### Class B — Uncommitted candidates pool

B2. **Per-location "candidates" buffer.** A second drop zone per location: the user can drag a Pokémon onto a location (without picking a house) to mark it a candidate for that area. They can later drag from the candidates buffer into a house, or remove it. This unlocks a "plan first, place later" workflow.

   *Data model:* extend the store with `candidatesByLocation: Record<LocationId, string[]>` (Pokémon ids).

   *UI:* a dropzone strip below the location tabs (or a "Candidates" panel that appears when the active location's candidates pool is non-empty). Pokémon in the candidates pool render as smaller chips; dragging from picker → candidates adds; dragging from candidates → a slot moves; dragging from candidates → trash icon removes.

   *Phase-4 hook:* this is a meaningful new structure — flag a Phase 5 plan to design it. The bootstrap should still ship without it.

### Class C — Theming + layout

C1. **Light/dark theme toggle.** The reference site is dark-only (with a Text Size S/M/L/XL menu). Add a theme toggle. *Phase-4 hook:* use CSS custom properties for all colors (the bootstrap plan already does this), so the toggle is a `data-theme="light|dark"` attribute on `<html>`.

C2. **Color scheme + minor layout adjustments.** Deferred — collect specifics after the bootstrap is deployed.

### Phase 5 ordering

When Phase 5 begins, the order is roughly: A2 (detail panel) → A1 (group toggle) → A3 (litter drops, after the user populates the TSV) → B1 (recommendations) → C1 (theme toggle) → B2 (candidates pool — biggest, deserves its own plan).
