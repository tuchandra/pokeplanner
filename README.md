# pokeplanner

A habitat planner for Pokémon Pokopia, deployed at **https://tusharc.dev/pokeplanner/**.

Drag (or click) Pokémon from the sidebar into prefab/custom houses across the five Pokopia areas. Houses auto-derive their lighting and feature-tag chips from the assigned Pokémon. State persists locally — there's no account or backend.

## Stack

- **bun** + **Vite 5** for tooling, **TypeScript** (strict) checked with **tsgo**
- **React 18** functional components
- **Tailwind v4** (Vite plugin, no PostCSS) + small hand-rolled shadcn-style primitives over **Radix UI** in `src/components/ui/`
- **Zustand** with `persist` middleware → `localStorage` key `pokeplanner.v1`
- **@dnd-kit/core** for drag-and-drop
- **Biome** for lint + format
- **bun:test** for unit tests
- Deployed via **GitHub Actions → GitHub Pages** with custom domain inherited from `tuchandra.github.io`

## Develop

    bun install
    bun run dev          # http://localhost:5173

    bun run typecheck    # tsgo --noEmit
    bun run lint         # biome check
    bun run format       # biome format --write
    bun test             # bun:test

    bun run build        # vite build → dist/
    bun run preview      # serve dist/

## Data ingest

The Pokémon roster lives in `src/data/pokemon.generated.ts`, and the various icon sets in `public/`. They're regenerated by scripts:

    bun run ingest             # poketracker.tsv + litter-drops-raw.txt → pokemon.generated.ts
    bun run mirror-sprites     # Pokémon sprites → public/sprites/<id>.png
    bun run mirror-litter      # Litter item icons → public/litter/<slug>.png  (+ litter-items.generated.ts)
    bun run mirror-locations   # Area icons → public/locations/<id>.png
    bun run mirror-specialties # Specialty icons → public/specialties/<slug>.png  (+ specialty-icons.generated.ts)

Each mirror script is idempotent — files already present are skipped. Inputs in `docs/reference/` (the user's TSV, raw litter-drops list, the saved reference site) are intentionally local-only and not committed.

## Layout

- `src/types.ts` — domain types (`House`, `Pokemon`, `Location`, `Specialty`, …) plus pure helpers `capacityPoints()` and `derivedHabitats()`.
- `src/data/` — typed game data: locations, lighting, specialties, tags, tastes, the Pokémon roster, and the per-name maps for litter items + specialty icons.
- `src/state/store.ts` — single Zustand store for houses, filters, and selection.
- `src/lib/` — pure helpers (`recommend`, `picker-groups`, `tsv`, `cn`).
- `src/components/` — feature components (`Topbar`, `HouseCard`, `HouseGrid`, `HouseTable`, `PokemonPicker`, `PokemonDetail`, `SpecialtyFilter`).
- `src/components/ui/` — local shadcn-style primitives (`Button`, `Popover`, `Select`, `ToggleGroup`, `Checkbox`).
- `scripts/` — one-shot ingest + mirror scripts.
- `docs/plans/` — implementation plan + status notes.
