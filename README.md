# pokeplanner

A habitat planner for Pokémon Pokopia, deployed at **https://tusharc.dev/pokeplanner/**.

Drag (or click) Pokémon from the sidebar into prefab/custom houses across the five Pokopia areas. Each house auto-derives its lighting and feature-tag chips from the assigned Pokémon. State persists in `localStorage` and can be shared across devices via a URL hash — no account or backend involved.

312 Pokémon ingested from the canonical TSV; 5 are story-only / Cloud-Island-exclusive and can't be housed (Tangrowth, Ho-Oh, Lugia, Volcanion, Kyogre), leaving 307 that need a home.

## Stack

- **bun** + **Vite 5** for tooling, **TypeScript** (strict) checked with **tsgo**
- **React 18** functional components
- **Tailwind v4** (Vite plugin, no PostCSS) + small hand-rolled shadcn-style primitives over **Radix UI** in `src/components/ui/`
- **Zustand** with `persist` middleware → `localStorage` key `pokeplanner.v1`
- **@dnd-kit/core** for drag-and-drop
- **Biome** for lint + format
- **bun:test** for unit tests
- Deployed via **GitHub Actions → GitHub Pages** with custom domain inherited from `tuchandra.github.io`

## Develop

    bun install
    bun run dev          # http://localhost:5173

    bun run typecheck    # tsgo --noEmit
    bun run lint         # biome check
    bun run format       # biome format --write
    bun test             # bun:test

    bun run build        # vite build → dist/
    bun run preview      # serve dist/

## Data ingest

The Pokémon roster lives in `src/data/pokemon.generated.ts`, and the various icon sets in `public/`. They're regenerated by scripts:

    bun run ingest             # poketracker.tsv → pokemon.generated.ts
    bun run mirror-sprites     # Pokémon sprites → public/sprites/<id>.png
    bun run mirror-litter      # Litter item icons → public/litter/<slug>.png  (+ litter-items.generated.ts)
    bun run mirror-locations   # Area icons → public/locations/<id>.png
    bun run mirror-specialties # Specialty icons → public/specialties/<slug>.png  (+ specialty-icons.generated.ts)

Each mirror script is idempotent — files already present are skipped. Inputs in `docs/reference/` (the user's TSV plus saved reference material) are intentionally local-only and not committed; the generated TS modules and PNGs are.

## Cross-device sharing

The Share button in the topbar gzips the planner state, base64url-encodes it, drops it in the URL hash, and copies the URL to your clipboard. Open that URL on any other device and the app hydrates from the hash on load (with a confirm prompt if you have an existing local planner).

## Mobile

- Topbar layout switches at the lg breakpoint (1024px). Below that, brand+actions sit on the first row and the location tabs get their own full-width row underneath (with hidden horizontal-scroll if all five don't fit). At lg+ everything sits on a single line.
- The Pokémon picker is hidden by default on screens narrower than md (768px) and slides in as a drawer when the user taps the panel-toggle button in the topbar. Backdrop tap or in-drawer × dismisses.
- House title is click-to-edit (pencil icon) so tapping anywhere on a card never accidentally focuses the input.
- House table view collapses location/type/slots into a single row of mono metadata under the name, with a destructive `×` icon (with confirm) replacing the wider "Delete" button.

## Layout

- `src/types.ts` — domain types (`House`, `Pokemon`, `Location`, `Specialty`, …) plus pure helpers `capacityPoints()` and `derivedHabitats()`.
- `src/data/` — typed game data: locations, lighting, specialties, tags, tastes, the Pokémon roster, and the per-name maps for litter items + specialty icons.
- `src/state/store.ts` — single Zustand store for houses, filters, and selection.
- `src/lib/` — pure helpers (`recommend`, `picker-groups`, `tsv`, `cn`).
- `src/components/` — feature components (`Topbar`, `HouseCard`, `HouseGrid`, `HouseTable`, `PokemonPicker`, `PokemonDetail`, `SpecialtyFilter`).
- `src/components/ui/` — local shadcn-style primitives (`Button`, `Popover`, `Select`, `ToggleGroup`, `Checkbox`).
- `scripts/` — one-shot ingest + mirror scripts.
- `docs/plans/` — implementation plan + status notes.
